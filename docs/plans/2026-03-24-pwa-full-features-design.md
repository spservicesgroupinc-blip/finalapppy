# PWA Full Features Design — RFE Foam Pro

**Date:** 2026-03-24
**Approach:** Option B — vite-plugin-pwa + Workbox

## Goal

Make RFE Foam Pro fully installable and offline-capable on desktop (Chrome/Edge) and smartphone (Android/iOS Safari), with all non-push PWA features enabled.

## Architecture

Replace the manual `public/sw.js` with a Workbox-powered service worker generated by `vite-plugin-pwa`. The plugin auto-generates a precache manifest at build time and injects it into the SW. Runtime caching strategies handle API and CDN assets.

## Components

### 1. vite.config.ts — Add VitePWA plugin
- Configure `registerType: 'prompt'` so updates show a toast instead of silent replace
- Workbox strategies: `CacheFirst` for static assets, `NetworkFirst` for navigation
- Inline SW (`inlineDynamicImports: true`) to avoid module issues
- Include `manifest.json` fields directly in plugin config

### 2. manifest.json — Fix icon and screenshot issues
- Replace all external CDN icon URLs with self-hosted SVG data URIs embedded via the plugin
- Add proper `180x180` apple-touch-icon entry
- Fix `start_url` to `/` (absolute, not relative)
- Add `prefer_related_applications: false`

### 3. index.html — iOS completeness
- Add `<link rel="apple-touch-icon" sizes="180x180">` pointing to `/apple-touch-icon.png`
- Add Apple splash screen `<link>` tags for common iOS device sizes
- Add Open Graph meta tags for share previews
- Keep existing WCO draggable title bar and iOS meta tags

### 4. public/icons/ — Local icon assets
- Generate a single local SVG icon and PNG variants (192, 512, 180) stored in `public/`
- Remove dependency on external flaticon CDN for icons

### 5. public/offline.html — Offline fallback page
- Minimal branded page shown when navigation fails and no cache hit exists
- Matches RFE color scheme (#0F172A dark blue)

### 6. components/PWAInstallPrompt.tsx — Install banner
- Listens for `beforeinstallprompt` event (Chrome/Edge/Android)
- iOS detection via `navigator.standalone` + Safari UA — shows manual instructions
- Dismissible banner at bottom of screen
- Stores dismissed state in localStorage

### 7. components/PWAUpdateToast.tsx — Update notification
- Uses `vite-plugin-pwa`'s `useRegisterSW` hook
- Shows toast "New version available — Reload" when SW update is waiting
- Calls `updateServiceWorker(true)` on confirm

### 8. components/NetworkStatus.tsx — Offline indicator
- Listens to `online`/`offline` window events
- Shows a slim top banner "You are offline" when disconnected
- Disappears automatically when reconnected

### 9. src/sw.ts — Custom Workbox service worker
- Precaches all Vite build output via `__WB_MANIFEST`
- Runtime cache: `StaleWhileRevalidate` for images/fonts
- Background Sync: queues failed POST requests (estimate saves) for retry
- `cleanupOutdatedCaches()` on activate
- Offline fallback: serve `/offline.html` on navigate failures

## Data Flow

```
App boots → SW registers → precache all assets
User goes offline → fetch intercept → serve from cache → show NetworkStatus banner
User has new data to save offline → Background Sync queue → retry on reconnect
New deploy → SW update found → PWAUpdateToast shown → user clicks Reload → new SW activates
User visits for first time → beforeinstallprompt fires → PWAInstallPrompt banner shown
```
